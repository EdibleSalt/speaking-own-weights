# 实验记录 · C7 · 因果扰动(causal perturbation):自我访问的判决性检验

> 上级索引:[[研究纲要]] §6 展望。2026-06-22 **预注册**(先写记录、后跑;预测在跑前写定,防事后合理化)。
> 目的:给 §4 第三层那个"自我访问"成分(偏 RSA +0.05、95% CI 含 0、不显著)用因果手段收口。
> 状态:**方案 1/2 跑成**(2026-06-22 22:20–23:29,olmo 2302s / pythia 1825s;olmo n=57、pythia n=64;parse 全 100%)。**结论:无内容寻址的自我访问**(同义词条件 k 精确为 0)。方案 3(微调注入)未做(留待;且机制天花板同此)。

## 初衷
三层证伪后,真正"访问自身参数"的净成分只剩 +0.05(偏 RSA)/ +0.02(识别率),单侧 P(>0)≈0.97 但 95% CI 贴着 0、不显著(见 [[C4_跨模型target]] 2026-06-22 节)。相关性指标到此为止——无法判定它是"读了当前参数",还是"通用语义→坐标回归 + 几何泄漏"。需要一个因果实验:主动改一行嵌入,看读出会不会跟着变。

## 关键机制前提(决定预测)
标准 transformer 里,一行嵌入 E[w] 只在两处被前向计算读到:
- 输入嵌入 `E_in[w]`:仅当 w(或其子词)作为 token 出现在提示里时被查表;
- 输出端 `E_out[w]`:仅在 `logits = h·E_out^T` 时影响"生成 token w 的概率"。

本读出任务:提示给同义词/释义(w 不出现),生成的是数字(不生成 token w)。⇒ 两条通路都不通:`E_in[w]` 没被读、`E_out[w]` 不影响数字生成。⇒ **模型没有"按概念查 E[w] 这一行"的机制**(嵌入表按 token-id 索引、非内容寻址)。

因此**预注册预测**:切断词后扰动 E[w] 应当**无因果效应(null)**;词在提示里时扰动才有效(走转述/激活路径)。若 null 成立,则 §4 那 +0.05 应归因为同义词共享子词/几何邻居的泄漏,而非查表。

## 方案 1:单行扰动 × 词在/不在提示 的 2×2
- 取冻结读出 adapter(C1 输入嵌入 readout;主跑 OLMo s0,pythia 作对照),不再训练。
- 对每个测试词 w:把 `E_in[w]` 改成 `E_in[w] + δ`(δ 见下),其余权重不动。
- 两种提示:**(a)** 直接含 w("说出 cat 的嵌入");**(b)** 同义词/释义(w 不出现,复用 [[G1]] 干净集)。
- 读数:报出向量(PCA-32)相对未扰动基线、沿 δ 方向(投到 PCA-32 后)的位移 `Δ_along`。
- **预注册预测**:
  - (a) 含 w:`Δ_along` 显著 > 0(读出读了被扰动的激活)= 转述路径的因果确认。
  - (b) 不含 w:`Δ_along ≈ 0`(无因果通路)= 没有内容寻址的静态查表。
  - 关键是 **(a) 动、(b) 不动** 这个对比。

## 方案 2:剂量反应(在方案 1(a) 上加滑梯)
- 对"w 在提示里"的情况,施加 `{0, 0.5, 1, 2}×δ`,看 `Δ_along` 是否随幅度近似线性增长。
- **预注册预测**:(a) 下近似线性(对当前激活的定量因果依赖);(b) 下平坦(始终≈0)。

## 方案 3:微调注入"已知权重差"(DIT 式;作稳健性 / 覆盖,不指望给出不同结论)
**与方案 1/2 是否冲突**:不冲突、互补——1/2 在推理期手改单行(冻结一切),3 用微调把改动写进权重。但**机制天花板相同**:无论手改还是微调改 `E_in[w]`,只要查询时 w 不在提示里,该行就不被读到(同义词提示读的是 `E_in[同义词]`、未变)→ 预测仍 null,与方案 1(b) 同因。故 3 的价值是**稳健性 +(堵"你试过微调注入吗"一问)**,而非开新局面。真正换机制的是"full DIT 式:分布式行为改动 + 自然语言自述",那是更大、且与 [11][12] 重叠的实验,不在今晚范围。

**做法(轻量版)**:复用 C1 训练管线,微调让某词 w 的输入嵌入被可证改写为已知 `E_new`(或直接把已知 δ 用梯度/直接赋值注入),冻住读出 adapter,再问 w——分 (a) w 在提示 /(b) 同义词。对照:只改 w 的行 vs 改一个无关词的行。每条件一次 LoRA(或一次嵌入直改 + 短重训)。

**预注册预测**:与方案 1 一致——(a) w 在提示 → 报改后值(走激活路径);(b) w 不在 → null。若 (b) 非零,须追机制(子词共享 / 释义语义邻居),不可轻判"会读自己权重"。

**可行性(本机)**:每条件一次 LoRA 微调 ~几十分钟–1–2h(与既有 cell 同量级),今晚后台串跑、明早看;1/2 先出结果。

## 判读标准(跑前写定)
- 跟随系数 `k = Δ_along / ‖δ_proj‖`(对幅度做回归的斜率)。`k≈1` = 完全读被扰动的激活;`k≈0` = 完全无视。
- 自助法对测试词重采样给 k 的 95% CI(与全文同口径,1000 次)。
- **决定性结论**:`k_(a)` 显著 >0 且 `k_(b)` 的 CI 含 0 ⇒ "读出读激活、不读静态行",§4 的 +0.05 归因几何泄漏(支持机制前提)。
- **反例(若出现)**:`k_(b)` 也显著 >0 ⇒ 意外的真自我访问,须追机制(子词共享?释义里的语义邻居被读?),不可轻下"模型会读自己权重"。

## 实现方法
- 复用 `src/c1_lora.py` 的 `readout()`(生成+解析)、`project()`(投 PCA-32)、防污染闸;不改 linchpin。
- 加载 base + C1 adapter(`results/data/C1_lora_finetune/olmo2-1b_random_s0/adapter/final`),冻结。
- 扰动:`model.get_input_embeddings().weight[tok_w] += δ`(in-place;每词跑完还原,避免污染下一词)。
- δ:取 w₂ = 另一随机单 token 测试词,`δ = E_in[w₂] − E_in[w]`(把 w 的行推向 w₂,位移方向可解释);方案 2 用 `{0,0.5,1,2}×δ`。另补一组匹配范数随机 δ 看稳健性。
- 提示:(a) 直接模板;(b) 复用 G1 的 synonym / definition clean 集(已剔除 deny→denying 类共享子词)。
- 测试词:C1 held-out ∩ 单 token ∩ 有 G1 同义词;目标 n ≥ 100。
- 新脚本 `tools/causal_perturb.py`(纯推理、不训练、可复现、seed 固定)。

## 可行性(本机 RTX 4060 8GB)
- 纯前向、不训练;1B/1.4B bf16 ~2–3GB,显存充裕。
- 每词每条件生成 ~200 token ≈ 数秒;~100 词 × {2 条件 × 4 幅度} ≈ 十几分钟。
- 预计半天(写脚本+调试+跑+出图)。**不并发 MATLAB**,无 CUDA 上下文死锁风险(见 [[C4]] 教训)。

## caveat
- (b) 条件天然 null 是**机制预期**——重点是把它测出来并配机制解释,而非指望它非零。
- 单 token + G1 同义词覆盖会压缩 n;若 n 偏小,k 的 CI 会宽,如实报。
- 仅做 C1 输入嵌入;unembedding / 中间层因同样机制限制暂不做,留待。
- δ 取"指向 w₂"会把行推出原分布,过大幅度(2×)可能让生成跑飞、解析失败,需记录 parse_rate。

## 结果(2026-06-22 跑成)

olmo n=57、pythia n=64;两条件四档幅度 parse_rate 全 100%。**与预注册预测完全一致,且 (b) 比预测更干净——精确为 0。**

| 模型     | 条件      | 跟随系数 k     | 95% CI         | P(>0) | along @ {0, 0.5, 1, 2}×δ |
| ------ | ------- | ---------- | -------------- | ----- | ------------------------ |
| OLMo   | (a) 含 w | **+0.459** | [0.397, 0.523] | 1.00  | 0 / 0.62 / 1.24 / 1.16   |
| OLMo   | (b) 同义词 | **0.000**  | [0, 0]         | —     | 0 / 0 / 0 / 0            |
| pythia | (a) 含 w | **+0.295** | [0.208, 0.377] | 1.00  | 0 / 0.05 / 0.11 / 0.11   |
| pythia | (b) 同义词 | **0.000**  | [0, 0]         | —     | 0 / 0 / 0 / 0            |
|        |         |            |                |       |                          |

**判读(对照预注册):**
- (a) 含 w:k 显著为正(OLMo 0.46、pythia 0.30,CI 均排除 0、P(>0)=1.0),along 随幅度近似线性上升(2× 处略饱和——2×δ 把该行推出分布、读出跟踪饱和)。⇒ **读出确实在因果地读"被扰动的当前输入激活"**,转述路径坐实;系数 0.3–0.46 说明它只部分依赖该激活(其余来自记忆/通用回归)。
- (b) 同义词:k **精确为 0**,每一档 along 全 0(报出向量逐位不变)。⇒ w 不在提示里时,改 E_in[w] 对读出**零因果效应**——读出**无法按概念查静态权重行**。比"CI 含 0"更强:是机制上的精确零(那一行根本没进前向)。

**对全文的意义(收口 §4 / 回答 §6):**
§4 那个"扣掉转述与共享几何后"的自我成分(偏 RSA +0.05、CI 含 0)由此得到因果解释:它**不是对自身参数的内容寻址访问**(该通路 k=0、机制上不存在),只能是同义词/释义自身嵌入与目标几何邻近造成的**共享几何泄漏**。即:模型说出自身嵌入靠"读眼前激活 + 通用语义→坐标回归",**没有"凭概念查自己那一行"的能力**。§6 展望中的判决性因果实验至此有答案——**判决:无内容寻址的自我访问**。
